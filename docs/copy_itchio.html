.copy — NextZXOS dot command for ZX Spectrum Next

A standalone dot command for NextZXOS / esxDOS that copies or moves files and entire directory trees. Run from the command line or from BASIC as .copy source dest.

Installation

Copy the file copy (no extension) into the /dot directory on your SD card.
Run it directly from the NextZXOS command line or from BASIC with the .copy command.

Syntax

.copy source dest
.copy -s source -d dest

Both forms are equivalent. Enclose paths that contain spaces in double quotes.

Options

  • -m — move: deletes the source file or directory after a successful copy
  • -y — yes: overwrites existing destination files; without this flag existing files are skipped
  • -h — show help

How it works

On startup the command automatically detects whether the source is a file or a directory by attempting to open it as an LFN directory.

Single file copy:
The file is read into an 8 kB work buffer at $DC00 and written to the destination in chunks. If the destination ends with a slash or is an existing directory, the source filename is appended automatically. Otherwise the destination path is used as the final filename.

Directory copy (recursive):

  1. The command traverses the directory tree up to 11 levels deep (MAX_DEPTH = 11).
  2. Each subdirectory is created at the destination (F_MKDIR — “already exists” errors are ignored).
  3. Files in each subdirectory are copied with long filenames (LFN) preserved.
  4. Before the actual copy a quick pre-scan counts all items to enable progress display.
  5. Progress is shown as Copying: dirname (X/total) and per-file as filename [XkB/TotalkB].
  6. Directory handles are closed before recursing and reopened afterwards — this works around the NextZXOS limit on simultaneously open handles.

Moving a directory (-m):
A full recursive copy is performed first, then the entire source tree is deleted (files via F_UNLINK, directories via F_RMDIR bottom-up). For a single file, the command tries F_RENAME first (fast same-drive rename); only if that fails does it fall back to copy + delete.

Overwrite behaviour

  • Without -y: an existing destination file is skipped and Skipping existing: name is shown.
  • With -y: the destination file is overwritten.

Destination path rules

  • Destination ends with / or \ → source name is appended after the slash.
  • Destination is an existing directory → source name is copied into that directory.
  • Otherwise → destination is used as the final file/directory name.
  • A trailing slash on the source path (dir/) is stripped so it is treated the same as dir.

Safety features

  • Nested-destination guard: the command checks whether the destination path is a subdirectory of the source and refuses to copy in that case.
  • Destination parent check: before starting, the command verifies that the parent directory of the destination exists. If not, it exits with copy: destiny path not found.
  • BREAK to cancel (CAPS SHIFT + SPACE): stops the operation at any time, deletes the partially written destination file, and prints Cancelled.
  • A ROM error handler is installed so that drive errors do not fall through to BASIC (common on some emulators with missing drives).

Performance

During copy and delete operations the CPU is temporarily switched to 28 MHz (NextReg $07 = 3). The original turbo setting is restored on exit, whether the command succeeds, fails, or is cancelled.

Technical details

  • Work buffer: 8192 bytes at $DC00–$FBFF
  • Source path stack: $C000 (12 × 256 bytes)
  • Destination path stack: $CC00 (12 × 256 bytes)
  • Maximum path length: 255 characters
  • Maximum subdirectory depth: 11 levels
  • Directories are read in LFN mode → long filenames are preserved
  • Loaded at address $2000 (standard dot command)

Examples

.copy backup backup2

Copies the file or directory backup to backup2.

.copy "Calm Commander" "c:/backup/"

Copies the directory Calm Commander (name contains a space) to c:/backup/Calm Commander.

.copy -s "my dir" -d "d:/my backup"

Copy using named flags — useful when paths contain spaces.

.copy -m olddir newdir

Moves the directory olddir to newdir (copy then delete source).

Messages

  • copy: missing or bad arguments — source or destination missing, or unknown flag
  • copy: destiny path not found — the parent directory of the destination does not exist
  • copy: esxDOS error $XX stg $YY — internal error; XX is the esxDOS error code, YY is the stage (for debugging)
  • Skipping existing: name — file exists at destination and -y was not given
  • Cancelled. — user pressed BREAK
  • Done. — copy or move completed successfully
