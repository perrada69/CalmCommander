.copy update: safer memory handling and operation summary

Small but important update to the .copy dot command.

Safer memory paging

.copy now saves the current MMU mapping and pages in dedicated work memory for its internal buffers instead of borrowing space that could overlap BASIC RAM or the system stack. The original mapping is restored before the command returns control — including when you cancel the operation with BREAK. This fixes a rare issue where pressing BREAK during a copy could leave the stack in a bad state.

Operation summary

After a copy (or move) finishes, .copy now prints a short summary: how many files and directories were actually copied. The same summary is shown if you cancel with BREAK, so you always know how far the operation got.

Part of the Calm Commander project.
